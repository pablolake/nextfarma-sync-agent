Conexión SQL Server (Farmatic)

Los valores por defecto funcionan en la mayoría de instalaciones. Usa "Más info" si necesitas verificar algún campo.

Dónde encontrarlo: Inicio → SQL Server Configuration Manager → SQL Server Network Configuration → Protocols for FARMATIC → TCP/IP → pestaña IP Addresses → IPAll.
Servidor: normalmente localhost o el nombre del PC.
Instancia: el nombre entre paréntesis en "SQL Server (INSTANCIA)".
Puerto: campo TCP Port (por defecto 1433, puede estar vacío si usa puerto dinámico).
SQL para verificar:
SELECT @@SERVERNAME AS servidor, @@SERVICENAME AS instancia,
       local_tcp_port AS puerto
FROM   sys.dm_exec_connections
WHERE  session_id = @@SPID
Usuario: normalmente sa (administrador de SQL Server).
Contraseña: establecida durante la instalación de Farmatic. Prueba con farmatic, el nombre de la farmacia, o consulta a tu proveedor de Farmatic.
Si no recuerdas la contraseña, tu proveedor de Farmatic puede restablecerla.
SQL para verificar (con acceso Windows Authentication):
SELECT name, type_desc, is_disabled
FROM   sys.server_principals
WHERE  type IN ('S','U') AND name NOT LIKE '##%'
ORDER  BY name
Farmatic: nombre de la base de datos principal, normalmente Farmatic.
Consejo (BOT+): base de datos del Consejo General de Farmacéuticos con grupos homogéneos y PVP menor. Normalmente Consejo. Si no existe, los grupos homogéneos no estarán disponibles pero el resto del sync funciona.
SQL para verificar (lista todas las bases de datos disponibles):
SELECT name, state_desc, recovery_model_desc
FROM   sys.databases
WHERE  name NOT IN ('master','tempdb','model','msdb')
ORDER  BY name
